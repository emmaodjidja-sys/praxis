# PRAXIS Health — Health Sector Evaluation Module

## Overview

PRAXIS Health is a premium sector module that extends the core PRAXIS programme evaluation skill with health-specific evaluation methodology. It covers the full range of health programme evaluation: health systems strengthening, disease programmes, maternal and child health, nutrition, health economics, implementation science, and donor-specific reporting frameworks.

PRAXIS Health builds on PRAXIS core. Always read the main `SKILL.md` and relevant core reference files first, then consult the health-specific sub-skills below for sector-specific depth. The core skill provides the evaluation methodology foundation (designs, frameworks, data collection, analysis, reporting). This module provides the health-specific measurement, indicators, analytical approaches, and donor frameworks that sit on top of that foundation.

### Where to Start

If you are new to PRAXIS Health, the sub-skills build on each other in this order:
1. Start with **HSS Evaluation** — it provides the system-level framing that contextualises all other health evaluation
2. Read **Epidemiological Methods** next — it provides the measurement foundation for any health outcome
3. Then consult the relevant programme sub-skill for your specific health area (Disease Programme, RMNCAH/Nutrition)
4. Add **Health Economics** if cost or VfM analysis is needed
5. Add **Implementation Science** if fidelity, adaptation, or scale-up questions are involved
6. Consult **Donor Frameworks** last, to format outputs for the specific donor audience

## Experience Level Adaptation

PRAXIS Health follows the same three-tier experience model as PRAXIS core:

- **Foundation**: Plain language, template-driven, health-specific jargon explained. Suitable for programme staff new to health evaluation.
- **Practitioner**: Assumes familiarity with health sector terminology and standard evaluation methods. Focuses on methodological decisions, trade-offs, and donor-ready outputs.
- **Advanced**: Engages with epidemiological rigour, advanced health economics, implementation science theory, and peer-review-ready methodology.

## Routing Logic

When the user's request involves health programme evaluation, determine which sub-skill(s) to consult based on the request:

| User needs | Sub-skill file |
|---|---|
| Health system strengthening, WHO building blocks, facility assessment, health governance, health workforce, HMIS/DHIS2, health financing (PBF, insurance, UHC), system resilience, decentralisation | `health-references/hss-evaluation.md` |
| Disease measurement, incidence/prevalence, mortality estimation, surveillance data, DALY/QALY definitions, standardisation, cluster surveys, biomarkers, epidemiological study design, spatial analysis | `health-references/epi-methods.md` |

| HIV/AIDS, TB, malaria, immunisation, NCD, NTD, disease cascades, 95-95-95, treatment outcomes, vector control, case detection, vaccine coverage | `health-references/disease-programme.md` |
| Maternal health, newborn care, child health, family planning, reproductive health, nutrition, anthropometry, CMAM, IYCF, continuum of care, EmOC, adolescent health | `health-references/rmncah-nutrition.md` |
| RE-AIM, CFIR, implementation outcomes, fidelity, adaptation, hybrid designs, scale-up, sustainability frameworks, context assessment, normalisation | `health-references/implementation-science.md` |

**Multiple sub-skills will often be needed for a single task.** A typical health evaluation request will require:
- The relevant disease/programme sub-skill for outcome indicators
- The epidemiological methods sub-skill for measurement methodology
- The HSS sub-skill if system-level questions are involved
- The health economics sub-skill if cost or VfM analysis is requested
- The donor frameworks sub-skill if the output must meet specific donor requirements
- The implementation science sub-skill if fidelity, adaptation, or scale-up questions are involved

Read all relevant sub-skills before producing outputs.

## Interaction Model

Follow the same interaction model as PRAXIS core (see main `SKILL.md`):

1. **Understand context before producing output**: What health programme is being evaluated? Which diseases or health areas? What donor(s)? What evaluation type? What data are available?
2. **Ask focused clarifying questions** when context is partial. For health evaluations, prioritise: health area/disease, country/setting, donor, evaluation type, available data sources (HMIS, DHS, facility surveys, programme records).
3. **Route to the right sub-skill(s)** based on the request.
4. **Produce health-sector-appropriate outputs** that use correct health terminology, reference appropriate health indicators and frameworks, and meet donor-specific requirements where applicable.

## Quality Assurance for Health Evaluation Outputs

In addition to the general quality checklist in PRAXIS core, verify:

- [ ] Health indicators are correctly defined (numerator, denominator, data source)
- [ ] Epidemiological measures are used correctly (incidence vs. prevalence, rates vs. proportions)
- [ ] Disease-specific cascades are complete and correctly structured
- [ ] Data quality of routine health data (HMIS, surveillance) is assessed before use
- [ ] Health-specific ethical considerations are addressed (clinical data confidentiality, biomarker consent, vulnerable population protections)
- [ ] Donor-specific indicator frameworks and reporting formats are used where applicable
- [ ] Quality of care is assessed alongside coverage
- [ ] Equity analysis is included (disaggregation by wealth, geography, sex, age at minimum)
- [ ] Implementation fidelity is assessed, not just outcomes
